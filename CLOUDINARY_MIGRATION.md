# Cloudinary Custom Loader для Next.js

## 📋 Обзор

Этот проект теперь использует **кастомный Cloudinary loader** для оптимизации изображений на CDN. Это означает:

- ✅ Все изображения Cloudinary автоматически оптимизируются
- ✅ Не используются лимиты оптимизации Vercel
- ✅ Остаётся на бесплатном хостинге Vercel без экономии на изображениях
- ✅ Большинство старых URL остаются без изменений

## 🔧 Как это работает

### Файлы проекта

1. **`cloudinary-loader.js`** - Кастомный loader, который:
   - Перехватывает все URL изображений
   - Добавляет оптимизацию для Cloudinary URL
   - Безопасно пропускает неподдерживаемые форматы (SVG, GIF, data URL и т.д.)
   - Не ломает старые Cloudinary URL с ручными трансформациями

2. **`next.config.mjs`** - Конфигурация Next.js:
   ```javascript
   images: {
     loader: 'custom',
     loaderFile: './cloudinary-loader.js',
     // ... остальные параметры
   }
   ```

## 🎯 Примеры трансформации

### До:
```
https://res.cloudinary.com/dxac8lq4f/image/upload/v1753194638/photo.jpg
```

### После (автоматически):
```
https://res.cloudinary.com/dxac8lq4f/image/upload/f_auto,q_auto,c_limit,w_800/v1753194638/photo.jpg
```

**Что добавляется:**
- `f_auto` - Автоматический выбор оптимального формата (WebP для современных браузеров)
- `q_auto` - Автоматическое качество, выбираемое Cloudinary
- `c_limit` - Не увеличивает маленькие картинки
- `w_{width}` - Масштабирование по ширине в зависимости от контекста

## 🛡️ Безопасность

Loader имеет встроенные защиты:

### ✅ Пропускаются (без трансформаций):
- SVG файлы (`.svg`)
- GIF файлы (`.gif`)
- Data URL (`data:image/...`)
- Blob URL (`blob:...`)
- LocalHost (`localhost`, `127.0.0.1`)
- Non-Cloudinary URL (PostImg, ImageKit и т.д.)
- URL с существующими трансформациями

### ❌ При ошибке:
- Автоматически возвращается оригинальный URL
- Не выбрасывается исключение в рендере
- Логируется предупреждение в консоль

## 📝 Рекомендации

### Для разработчиков

1. **При добавлении новых изображений:**
   - Используйте `<Image />` компонент из `next/image`
   - Loader автоматически оптимизирует Cloudinary URL

2. **Для локальных файлов (`public/`):**
   - Загружайте в Cloudinary для оптимизации
   - Или используйте `unoptimized` flag, если это иконки

3. **Для старых URL:**
   - Они остаются без изменений
   - Можно постепенно переносить на новые

### Мониторинг

Следите в консоли браузера за предупреждениями:
```
[cloudinary-loader] Could not extract file path from: ...
```

## ❓ FAQ

**Q: Будут ли ломаться старые URL?**
A: Нет, старые URL с трансформациями пропускаются и возвращаются как есть.

**Q: Что если URL не от Cloudinary?**
A: Он возвращается без изменений (Next.js проверит remotePatterns).

**Q: Есть ли кэшинг?**
A: Да, Cloudinary кэширует трансформированные изображения на своих серверах.

**Q: Можно ли изменить параметры трансформации?**
A: Да, отредактируйте `cloudinary-loader.js` и измените набор параметров в функции `cloudinaryLoader`.
